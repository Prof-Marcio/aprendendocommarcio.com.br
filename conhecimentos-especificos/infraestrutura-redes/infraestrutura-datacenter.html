Infraestrutura de Datacenter

Disciplina 2.2: Infraestrutura de TI e Redes | Conteúdo 2.2.1

🔎 Perfil da Banca — Fundação Carlos Chagas (FCC)

Estilo de cobrança em TI: Questões conceituais + aplicação prática. Foco em componentes, funções e classificação.

Tipo de raciocínio exigido: Identificar função de cada elemento do datacenter; Diferenciar camadas de infraestrutura (física x lógica).

Padrões recorrentes: redundância e alta disponibilidade, UPS (no-break), climatização, virtualização, tiers de datacenter.

Erros comuns: confundir backup com alta disponibilidade; achar que datacenter é só servidor; não entender redundância.

📚 NÍVEL 1 — VDG (Velho de Guerra)

🖥️ Infraestrutura de Datacenter

📌 1. Conceito (o que é)

Datacenter é um ambiente físico e lógico projetado para: 👉 armazenar, processar e gerenciar dados e sistemas. É onde ficam: servidores, redes e sistemas críticos.

🎯 2. Aplicação / características

Usado para: hospedar sistemas institucionais, armazenar dados, garantir disponibilidade de serviços e executar aplicações críticas.
📌 Em órgãos públicos (como MP), garante: continuidade dos serviços, segurança da informação e disponibilidade dos sistemas.

🧩 3. Estrutura ou elementos principais (Pareto FCC)

  • 🔹 1. Servidores: Responsáveis por processamento e execução de aplicações.
  • 🔹 2. Storage (armazenamento): Armazena dados, backups e bancos de dados (Ex: SAN, NAS).
  • 🔹 3. Rede: Componentes como switches, roteadores e firewalls. Função: 👉 comunicação entre sistemas.
  • 🔹 4. Energia elétrica: Inclui UPS (no-break) e geradores. Função: 👉 evitar parada do sistema.
  • 🔹 5. Climatização: Controle de temperatura e umidade. Função: 👉 evitar superaquecimento.
  • 🔹 6. Segurança física: Inclui controle de acesso, câmeras e monitoramento.
  • 🔹 7. Virtualização: Permite rodar vários sistemas em um único servidor (Ex: máquinas virtuais).
  • 🔹 8. Redundância: Duplicação de componentes críticos (energia, rede, servidores). Função: 👉 evitar falhas (alta disponibilidade).

⚙️ 4. Como funciona

Fluxo simplificado: 1️⃣ usuário acessa sistema -> 2️⃣ requisição chega ao datacenter -> 3️⃣ servidor processa -> 4️⃣ storage fornece dados -> 5️⃣ rede entrega resposta. Tudo com: energia contínua, refrigeração e segurança.

🎯 5. Como a FCC costuma cobrar

A Fundação Carlos Chagas cobra: função de cada componente, conceito de alta disponibilidade, diferença entre backup e redundância, papel da UPS e climatização.
Exemplo: O equipamento responsável por manter energia em caso de falha é: Resposta: UPS (no-break)

🟢 6. Exemplo simples

Se faltar energia: ➡️ o UPS mantém o funcionamento imediato; ➡️ o gerador assume depois.

🧠 7. Dica de memorização (Pareto FCC)

📌 UPS → energia contínua | 📌 climatização → evitar calor | 📌 redundância → evitar falha | 📌 virtualização → vários sistemas em um servidor.
Resumo: 🖥 servidor = processa | 💾 storage = guarda | 🌐 rede = conecta

⚫ NÍVEL 2 — Pegadinhas da banca (FCC)

  • 🔴 Backup garante alta disponibilidade → 🔵 Errado. Backup serve para recuperação de dados, não para manter o sistema ativo.
  • 🔴 Redundância é a mesma coisa que backup → 🔵 Errado. Redundância garante continuidade do serviço; backup garante recuperação após falha.
  • 🔴 UPS mantém o sistema funcionando por tempo indefinido → 🔵 Errado. O UPS mantém por curto período, até o gerador assumir ou desligamento seguro.
  • 🔴 Datacenter é composto apenas por servidores → 🔵 Errado. Inclui energia, rede, climatização, segurança e storage.
  • 🔴 Virtualização aumenta o número de servidores físicos → 🔵 Errado. Ela reduz a necessidade de servidores físicos, otimizando recursos.

⚫ NÍVEL 3 — Identificação rápida na prova (FCC)

  • 🔴 energia em caso de falha elétrica → 🟢 “Equipamento que mantém o sistema ligado imediatamente...” → 🔵 UPS (no-break)
  • 🔴 armazenamento de dados → 🟢 “Sistema responsável por guardar arquivos e bancos de dados” → 🔵 Storage (SAN/NAS)
  • 🔴 processamento de aplicações → 🟢 “Máquina que executa sistemas e serviços” → 🔵 Servidor
  • 🔴 controle de temperatura → 🟢 “Sistema que evita superaquecimento dos equipamentos” → 🔵 Climatização
  • 🔴 continuidade do serviço sem interrupção → 🟢 “Uso de componentes duplicados para evitar falhas” → 🔵 Redundância
  • 🔴 execução de múltiplos sistemas em um único hardware → 🟢 “Uso de máquinas virtuais...” → 🔵 Virtualização
  • 🔴 comunicação entre dispositivos → 🟢 “Equipamentos que interligam servidores e sistemas” → 🔵 Rede (switch/roteador)

⚫ NÍVEL 4 — Mapa mental final (5 segundos)

🔴 energia imediata → 🔵 UPS 🔴 processa aplicações → 🔵 Servidor 🔴 armazenamento de dados → 🔵 Storage 🔴 interliga dispositivos → 🔵 Rede 🔴 evitar superaquecimento → 🔵 Climatização 🔴 continuidade sem falha → 🔵 Redundância 🔴 vários sistemas em um servidor → 🔵 Virtualização
