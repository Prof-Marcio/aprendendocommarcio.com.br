🔐 Normas ISO de Segurança da Informação (Família ISO 27000)

As normas da família International Organization for Standardization ISO 27000 tratam da gestão da segurança da informação nas organizações. Elas orientam como proteger dados contra: acesso não autorizado, alteração indevida, perda ou indisponibilidade.

📌 1. Conceito (o que é)

As normas ISO 27000 definem boas práticas e requisitos para gestão da segurança da informação por meio do SGSI.
SGSI = Sistema de Gestão de Segurança da Informação.
Objetivo: Proteger informações garantindo a Tríade CIA:

  • 🔐 Confidentiality → confidencialidade
  • 🧩 Integrity → integridade
  • ⚙ Availability → disponibilidade

🎯 2. Aplicação / características

Essas normas são utilizadas para: proteger dados organizacionais; gerenciar riscos de segurança da informação; implementar controles de segurança; criar políticas de segurança. São aplicadas em: empresas privadas, órgãos públicos, bancos, provedores de tecnologia e órgãos governamentais.

🧩 3. Estrutura ou elementos principais (Pareto FCC)

🔹 ISO 27001: Define requisitos para implementação do SGSI. Características: norma certificável; estabelece políticas de segurança; define gestão de risco; exige melhoria contínua.

🔹 ISO 27002: Guia de boas práticas de segurança da informação. Contém catálogo de controles de segurança e orientações para proteção da informação. Exemplos: controle de acesso, criptografia, gestão de incidentes, segurança física, segurança de redes. Não é certificável.

🔹 ISO 27005: Norma específica de gestão de riscos em segurança da informação. Define: identificação de riscos, análise de riscos e tratamento de riscos.

🔹 ISO 27000: Norma que apresenta conceitos, terminologia e visão geral da família ISO 27000.

⚙️ 4. Como funciona (SGSI)

Implementação típica: 1️⃣ identificar ativos de informação; 2️⃣ avaliar riscos de segurança; 3️⃣ selecionar controles de segurança; 4️⃣ implementar políticas e procedimentos; 5️⃣ monitorar e melhorar continuamente. O SGSI utiliza o ciclo: 📊 PDCA (Plan, Do, Check, Act).

🎯 5. Como a FCC costuma cobrar

A Fundação Carlos Chagas costuma perguntar: qual norma define requisitos do SGSI; qual norma traz boas práticas; qual trata de gestão de risco; qual apresenta conceitos da família ISO 27000.
Exemplo típico: "A norma que estabelece requisitos para certificação do Sistema de Gestão de Segurança da Informação é..." Resposta: ISO 27001.

🟢 6. Exemplo simples

Uma empresa deseja obter certificação internacional em segurança da informação. Ela implementa o SGSI conforme ➡ ISO 27001. Para selecionar controles de segurança utiliza ➡ ISO 27002.

🧠 7. Dica de memorização (Pareto FCC)

  • 📜 27001 → requisitos + certificação
  • 📘 27002 → controles + boas práticas
  • 📊 27005 → gestão de riscos
  • 📖 27000 → conceitos da família

⚫ NÍVEL 2 — Pegadinhas da banca (FCC)

  • 🔴 ISO 27002 é certificável → 🔵 Errado. Apenas a ISO 27001 permite certificação do SGSI.
  • 🔴 ISO 27001 é apenas um guia de boas práticas → 🔵 Errado. A ISO 27001 define requisitos formais para implementar o SGSI.
  • 🔴 ISO 27005 define controles de segurança → 🔵 Errado. A ISO 27005 trata da gestão de riscos da informação.
  • 🔴 ISO 27000 estabelece requisitos para certificação → 🔵 Errado. A ISO 27000 apresenta conceitos e terminologia da família 27000.
  • 🔴 Implementar controles elimina completamente os riscos → 🔵 Errado. Controles servem para reduzir ou tratar riscos, não eliminá-los totalmente.
